# SmartReliability AI

**Предиктивная диагностика флотомашин и насосного оборудования**

Интеллектуальная система предиктивной диагностики динамического оборудования обогатительных фабрик на основе вибрационных, токовых и технологических сигналов. Система выявляет дефекты до отказа, рассчитывает индекс технического состояния, оценивает риск и формирует рекомендации для службы ремонта.

Решение разработано в **Научно-Инновационном Центре «Big Data & Blockchain Technologies», Astana IT University**.
Кейс-ориентир: ТОО «Корпорация Казахмыс».

---

## Возможности

- **Гибридный детектор аномалий** — контрольные карты (3σ) + One-Class SVM
- **Классификация дефектов** — Random Forest, 7 типов неисправностей
- **Health Index** — индекс технического состояния (0–100)
- **Оценка риска отказа** — на 7, 14, 30 дней
- **Частотный анализ вибрации (FFT)** — выявление дефектов подшипников по спектру
- **Live-симулятор** — потоковая диагностика в реальном времени
- **Валидация модели** — метрики, confusion matrix, раннее предупреждение (lead time)
- **Человек в контуре** — подтверждение диагноза инженером с сохранением в журнал
- **Схема промышленного внедрения** — от датчиков до ремонтной заявки

## Диагностируемые дефекты

`bearing_wear` (износ подшипника), `imbalance` (дисбаланс), `cavitation` (кавитация), `overload` (перегрузка), `clogging` (засорение), `motor_fault` (дефект двигателя), `normal` (норма).

## Метрики (на демонстрационных данных)

| Метрика | Значение |
|---|---|
| Обнаружение дефектов | 100% |
| Точность типа дефекта | ~78% |
| Macro-F1 | ~0.72 |
| ROC-AUC (детекция) | ~0.88 |
| Раннее предупреждение | ~48 ч до отказа |

## Установка и запуск

Требуется Python 3.11+.

```bash
# клонировать репозиторий
git clone https://github.com/Arailym-ray/SmartReliabilityAI.git
cd SmartReliabilityAI

# установить зависимости
pip install -r requirements.txt

# запустить дашборд
streamlit run app.py
```

Дашборд откроется в браузере по адресу `http://localhost:8501`.

## Вкладки дашборда

1. **Обзор** — стартовый экран с ключевыми показателями
2. **Парк оборудования** — мнемосхема цеха, ранжирование по риску
3. **Диагностика агрегата** — карточка агрегата, FFT, обратная связь инженера
4. **Отчёты** — экспорт в Excel, распределение парка
5. **Симулятор (live)** — потоковая диагностика в реальном времени
6. **Валидация модели** — метрики качества и раннее предупреждение
7. **Внедрение** — архитектура интеграции, аппаратная часть, специфика оборудования

## Данные

Демонстрационный прототип обучен на синтетических данных с реалистичными сигнатурами дефектов. Метрики не завышены, что подтверждает корректность пайплайна. Для промышленного применения требуется калибровка на реальных сигналах предприятия. Подробнее — в `docs/DATASET_CARD.md`.
