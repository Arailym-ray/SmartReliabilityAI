# SmartReliability AI

**Предиктивная диагностика флотомашин и насосного оборудования**

Интеллектуальная система предиктивной диагностики динамического оборудования обогатительных фабрик на основе вибрационных, токовых и технологических сигналов. Система выявляет дефекты до отказа, рассчитывает индекс технического состояния, оценивает риск и формирует рекомендации для службы ремонта.

Решение разработано в **Научно-Инновационном Центре «Big Data & Blockchain Technologies», Astana IT University**.
Кейс-ориентир: ТОО «Корпорация Казахмыс».

---

## Возможности

- **Гибридный детектор аномалий** — контрольные карты (3σ) + One-Class SVM, пороги калибруются по здоровым данным
- **Классификация дефектов** — Random Forest, 7 типов состояний
- **Health Index** — индекс технического состояния (0–100)
- **Оценка риска отказа** — на 7, 14, 30 дней
- **Частотный анализ вибрации (FFT)** — выявление дефектов подшипников по спектру
- **Live-симулятор** — потоковая диагностика в реальном времени со схемой флотомашины FM-8
- **Честная валидация** — метрики с классом «норма», confusion matrix, раннее предупреждение (lead time)
- **Человек в контуре** — подтверждение диагноза инженером с сохранением в журнал (CSV)
- **Схема промышленного внедрения** — от датчиков до ремонтной заявки, аппаратная часть, специфика оборудования

## Диагностируемые дефекты

`bearing_wear` (износ подшипника), `imbalance` (дисбаланс), `cavitation` (кавитация), `overload` (перегрузка), `clogging` (засорение), `motor_fault` (дефект двигателя), `normal` (норма).

## Метрики (на демонстрационных данных)

Оценка проводится на равном числе дефектных и здоровых сценариев (класс «норма»), поэтому метрики детекции учитывают ложные тревоги.

| Метрика | Значение |
|---|---|
| Accuracy детекции (с классом норма) | ~97% |
| Recall (полнота обнаружения) | 100% |
| Precision (точность тревог) | ~95% |
| Частота ложных тревог | ~6% |
| Точность типа дефекта | ~78% |
| Macro-F1 (классификация) | ~0.72 |
| Раннее предупреждение (lead time) | ~44 ч до отказа |

Матрица: TP=18, FN=0 (ни один дефект не пропущен), TN=17, FP=1 (низкая доля ложных тревог).

## Установка и запуск

Требуется Python 3.11+.

```bash
# клонировать репозиторий
git clone https://github.com/Arailym-ray/SmartReliabilityAI.git
cd SmartReliabilityAI

# установить зависимости
pip install -r requirements.txt

# запустить дашборд
streamlit run app.py
```

Дашборд откроется в браузере по адресу `http://localhost:8501`.

## Вкладки дашборда

1. **Обзор** — стартовый экран с ключевыми показателями
2. **Парк оборудования** — мнемосхема цеха, ранжирование по риску
3. **Диагностика агрегата** — карточка агрегата, FFT, обратная связь инженера
4. **Отчёты** — экспорт в Excel, распределение парка
5. **Симулятор (live)** — потоковая диагностика, схема флотомашины FM-8 с подсветкой датчиков дефекта
6. **Валидация модели** — метрики с классом «норма», confusion matrix, раннее предупреждение
7. **Внедрение** — архитектура интеграции, аппаратная часть, специфика оборудования (насосы vs флотомашины)

## Данные

Демонстрационный прототип обучен на синтетических данных с реалистичными сигнатурами дефектов. Метрики не завышены, что подтверждает корректность пайплайна. Для промышленного применения требуется калибровка порогов и переобучение на реальных сигналах предприятия. Подробнее — в `docs/DATASET_CARD.md`.

## Известные ограничения

- Данные синтетические; для внедрения нужна перекалибровка на реальных сигналах.
- Засорение и износ подшипника имеют близкие вибрационные сигнатуры и могут смешиваться в классификации (детекция аномалии при этом устойчива).
- Обратная связь инженера сохраняется в файл; автоматическое дообучение модели — задача следующего этапа.
- FFT реализован для агрегата с сырыми вибро-данными; для остальных нужны высокочастотные датчики.
