# SmartReliability AI — MVP

Интеллектуальная система предиктивной диагностики флотомашин и насосного
оборудования. Полный прототип по ТЗ: от загрузки данных до дашборда с ИИ.

Разработано в Astana IT University. Заказчик-ориентир: ТОО «Корпорация Казахмыс».

## Быстрый запуск

```bash
cd mvp
pip3 install -r requirements.txt
streamlit run app.py
```

Если на Mac несколько версий Python и streamlit «не найден», используйте ту,
куда установились пакеты, например: `python3.11 -m streamlit run app.py`.

Дашборд откроется в браузере (http://localhost:8501). Первый запуск ~10 секунд —
обучаются ИИ-модели.

## Структура

```
mvp/
├── app.py                  # Streamlit-дашборд (ТЗ 11-14)
├── requirements.txt
├── assets/                 # логотипы AITU и Kazakhmys
├── data/                   # датасет (CSV)
│   └── synthetic_sensor_timeseries.csv   # основная телеметрия (25 920 записей)
└── src/
    ├── data.py             # загрузка + предобработка (ТЗ 9.1, 9.2)
    ├── features.py         # извлечение признаков (ТЗ 9.3)
    ├── models.py           # ИИ-ядро: гибридный детектор + классификатор (ТЗ 10)
    ├── simulator.py        # live-симулятор потоковых данных
    ├── diagnostics.py      # рекомендации + экономика (ТЗ 12, 13)
    └── export.py           # экспорт отчётов в Excel (ТЗ 14)
```

## Где ИИ

В `src/models.py`, класс `DiagnosticEngine` — два алгоритма машинного обучения:

1. **Гибридный детектор аномалий** = контрольные карты (3σ, прозрачный контур)
   + One-Class SVM (ML-контур). Даёт Health Index и 4 уровня тревоги.
   ROC-AUC ≈ 0.88 на детекции дефектов, 0.99 против экспертных меток.
2. **Random Forest** — классификация типа дефекта (7 типов) с уверенностью.

Модели обучаются на данных при запуске (те ~10 секунд загрузки).

## Покрытие ТЗ

| Раздел ТЗ | Реализация |
|---|---|
| 9.1 Загрузка данных | `data.load_*` + загрузка своего CSV в дашборде |
| 9.2 Предобработка | `data.preprocess` (пропуски, выбросы, режимы) |
| 9.3 Признаки | `features` (тренды, перекос фаз, crest factor) |
| 10.1 Аномалии (4 уровня) | гибридный детектор → normal/warning/anomaly/critical |
| 10.2 Health Index (0-100) | `models` → hi |
| 10.3 Риск (7/14/30д) | `models` → risk_7/14/30 |
| 10.4 Классификация дефекта | Random Forest + уверенность |
| 11 Дашборд | `app.py` — 4 вкладки |
| 12 Рекомендации | `diagnostics` + объяснимость |
| 13 Экономический эффект | `diagnostics.economic_impact` |
| 14 Экспорт | Excel-отчёты (`export.py`) |

## Дашборд: четыре вкладки

1. **Парк оборудования** — все агрегаты по Health Index, уровень + пик тревоги 14д.
2. **Диагностика агрегата** — карточка: HI, дефект, риск, объяснение,
   рекомендация, графики вибрации/тока/температуры, HI во времени,
   аварии SCADA, экспорт детального отчёта.
3. **Экономический эффект** — предотвращённый простой, распределение парка,
   история отказов, экспорт отчёта по парку в Excel.
4. **Симулятор (live)** — имитация живого оборудования: выбираете дефект,
   данные текут в реальном времени, HI падает, срабатывают тревоги.

## Загрузка своих данных

В боковой панели — загрузчик CSV. По умолчанию работает тестовый датасет.
Свой файл требует минимум колонки: `timestamp`, `equipment_id`,
`vibration_rms`, `vibration_peak`, `current_a`, `temperature`, `flow_rate`,
`rpm`. Недостающие дозаполняются. Если данные без разметки дефектов — система
переходит в unsupervised-режим автоматически.

## Важно

Тестовые данные синтетические, для демонстрации архитектуры и логики. Для
промышленного применения модели и пороги требуют калибровки на реальных
сигналах предприятия.
