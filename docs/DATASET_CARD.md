## Структура данных

**Основная телеметрия** (`synthetic_sensor_timeseries.csv`) — 25 920 часовых записей, 28 колонок. 11 сенсорных сигналов + служебные метки для оценки.

**Сырая вибрация** (`synthetic_raw_vibration_PUMP-204.csv`) — 120 000 точек ускорения при 20 кГц, 6 снапшотов от здорового состояния до отказа. Для частотного (FFT) анализа.

**Справочники** — реестр оборудования (12), история ремонтов (44), история отказов (10), аварии SCADA (980).

## Смоделированные состояния

Нормальная работа, постепенный износ подшипника, дисбаланс, перегрузка двигателя, кавитация, засорение, рост вибрации перед отказом, аварийная остановка, восстановление после ремонта.

## Разделение на обучение/тест

Валидация проводится двумя протоколами для честной оценки:
- **GroupKFold по агрегату** — проверка обобщения на новый агрегат (строгий тест)
- **Временной split** (70/30) — реалистичный сценарий: обучение на прошлом, прогноз на будущем известного агрегата

Служебные колонки (метки, Health Index, риски) исключены из признаков модели — используются только как эталон для оценки (защита от утечки данных).

## Ограничения

Данные **синтетические**, сгенерированы с контролируемыми сигнатурами дефектов. Предназначены для демонстрации архитектуры, логики и интерфейса системы. **Не откалиброваны для промышленного применения** — для внедрения на реальном предприятии требуется калибровка на реальных сигналах оборудования.

Метрики на синтетике (ROC-AUC ~0.88, F1 ~0.72) реалистичны и не завышены, что подтверждает корректность пайплайна. На реальных данных предприятия ожидается адаптация порогов и переобучение моделей.

## Лицензия и назначение

Синтетические данные, создано для НИОКР-прототипа предиктивной диагностики флотомашин и насосного оборудования (кейс ТОО «Корпорация Казахмыс», Astana IT University).
